## `cap_write.asm:cap_write`

**summary:** capability `write` — emit bytes to stdout; buffer+len in, bytes-written out.

**detail:**

```text
Mandatory capability (DESIGN §4). One concept, two hand-written realizations selected by the -D WIN64 build flag.
· win64 → GetStdHandle(STD_OUTPUT_HANDLE=-11) then WriteFile(h, buf, len, &written, NULL); drags the Win64 call ABI (shadow space, RCX/RDX/R8/R9, arg5 on stack). Never issues syscall directly (numbers private).
· linux → syscall write(STDOUT_FD=1, buf, len); number 1 in RAX, args RDI/RSI/RDX.
Contract: arg1=buffer (rcx/rdi), arg2=byte count (rdx/rsi); returns bytes written in RAX. Written as a real function (own prologue/epilogue) so it can be promoted to a testable boundary (DESIGN §6.2).
```

## `abi:arg4`

**summary:** logical fourth argument — where the two conventions diverge most visibly.

**detail:**

```text
win64 → r9; sysv/linux → rcx. Note the collision: SysV arg4 lands in the register Win64 uses for arg1, so a body written to the wrong convention still assembles and runs — only behavior betrays it. For raw Linux syscalls arg4 is R10 instead (see abi:syscall-args); rcx applies to the *function-call* convention. Lineage: the r10 substitution is the one hardware-forced cell in the whole table — the syscall insn hardwires rcx←rip, r11←rflags — and Windows makes the same move privately (every ntdll stub is mov r10, rcx; syscall). See docs/abi-lineage.md.
```

## `abi:callee-saved`

**summary:** registers a callee must preserve; nadir stashes cross-call state in the r12–r15 intersection.

**detail:**

```text
win64 → rbx, rbp, rsp, rsi, rdi, r12–r15, xmm6–15; sysv/linux → rbx, rbp, rsp, r12–r15 (rsi/rdi are ARGUMENT registers there, xmm all volatile). The intersection — rbx, rbp, r12–r15 — is the only set that reads identically in both realizations, which is why m1_fold carries a..d and its result in r12–r15: one register choice, two ABIs, zero %ifdef in the stash logic. Preservation is by push/pop in the prologue/epilogue. Lineage: win64's larger set is continuity with x86-32 Windows (esi/edi were callee-saved there); sysv freed rsi/rdi to serve as arg1/arg2 and left all xmm volatile for cheap vector leaves. See docs/abi-lineage.md.
```

## `u64_to_dec.asm:u64_to_dec`

**summary:** u64 → decimal ASCII; value+buf_end in, pointer-to-first-digit out.

**detail:**

```text
M1 helper, almost portable-stratum (DESIGN §2.1): the divide loop is pure registers, identical on both targets; only the two arg-pickup movs are %ifdef'd (arg1=value: rcx/rdi, arg2=buf_end: rdx/rsi). Digits are generated least-significant-first walking backward from buf_end; returns rax = first digit, so digits occupy [rax, buf_end). Clobbers only rax/rdx/r8/r9 — volatile under BOTH conventions, so the shared body violates neither callee-saved set. Caller sizes the buffer (20 digits worst case) and owns any trailing newline. Loop runs at least once so 0 renders as '0'.
```

## `m1_fold.asm:m1_fold`

**summary:** M1 proof kernel — non-leaf, 4 args: fold(a,b,c,d)=a·1000+b·100+c·10+d, printed then returned.

**detail:**

```text
DESIGN §7.2 'prove the ABI stratum.' Deliberately ASYMMETRIC: every arg permutation yields different digits, so a swapped arg register in either realization fails the behavioral test — fold(6,7,8,9)=6789 literally spells the argument order. Pins what M0 could not: arg3/arg4 (win64 r8/r9 vs sysv rdx/rcx), and callee-saved discipline in both directions — preserves r12–r15 for its caller (push/pop) while RELYING on them to carry a..d and the result across three inner calls (cap_write, u64_to_dec, cap_write). Frame: win64 sub rsp,40 after 4 pushes (entry ≡8 → ≡0 at calls, 32B shadow homed); sysv sub rsp,8 (realign only, no shadow). Rendering scratch dec_buf(21B): digits end at +20, newline lives at [+20], one write covers digits+newline. Returns fold in rax.
```

## `m1_abi.asm:_start`

**summary:** M1 proof program — calls m1_fold(6,7,8,9); exit code is the ABI verdict (0 = stratum held).

**detail:**

```text
Converts two ABI facts into observable behavior, identical expectations both targets: (1) return value — exit code starts as rax−6789, zero iff the fold came back correct; the *diff* form keeps linux's 8-bit exit truncation moot. (2) callee-saved preservation — plants canary 0xC0FFEE in r12 before the call; m1_fold uses r12 internally, so a surviving canary proves its push/pop is real; a dead one forces exit 99. Entry duties per docs/asm-debugging-guide.md: win64 sub rsp,40 once (loader enters ≡8, frame never unwound since cap_exit doesn't return); sysv enters ≡0 so a bare call is aligned. Tested by tests/test_m1.py: exact stdout + exit 0.
```
