## `abi:arg4`

**summary:** logical fourth argument — where the two conventions diverge most visibly.

**detail:**

```text
win64 → r9; sysv/linux → rcx. Note the collision: SysV arg4 lands in the register Win64 uses for arg1, so a body written to the wrong convention still assembles and runs — only behavior betrays it. For raw Linux syscalls arg4 is R10 instead (see abi:syscall-args); rcx applies to the *function-call* convention. Lineage: the r10 substitution is the one hardware-forced cell in the whole table — the syscall insn hardwires rcx←rip, r11←rflags — and Windows makes the same move privately (every ntdll stub is mov r10, rcx; syscall). See docs/abi-lineage.md.
```

## `abi:callee-saved`

**summary:** registers a callee must preserve; nadir stashes cross-call state in the r12–r15 intersection.

**detail:**

```text
win64 → rbx, rbp, rsp, rsi, rdi, r12–r15, xmm6–15; sysv/linux → rbx, rbp, rsp, r12–r15 (rsi/rdi are ARGUMENT registers there, xmm all volatile). The intersection — rbx, rbp, r12–r15 — is the only set that reads identically in both realizations, which is why m1_fold carries a..d and its result in r12–r15: one register choice, two ABIs, zero %ifdef in the stash logic. Preservation is by push/pop in the prologue/epilogue. Lineage: win64's larger set is continuity with x86-32 Windows (esi/edi were callee-saved there); sysv freed rsi/rdi to serve as arg1/arg2 and left all xmm volatile for cheap vector leaves. See docs/abi-lineage.md.
```

**summary:** M1 proof kernel — non-leaf, 4 args: fold(a,b,c,d)=a·1000+b·100+c·10+d, printed then returned.

**detail:**

## `abi:nadir-call`

**summary:** nadir's internal call convention — one static register contract, both targets; the seam translates.

**detail:**

```text
Adopted v0.3 (DESIGN §2.2) after M1's two-realization build surfaced the seam as the right absorption point; codified in src/nadir.inc. args = rdi, rsi, rdx, rcx (arg5+ undefined until a capability needs it — pull-based); return = rax; callee-saved = rbx, rbp, r12–r15 (the win64∩sysv intersection, so kernel32 calls and syscalls preserve it for free); volatile = everything else incl. all xmm (the union); rsp ≡ 0 (mod 16) at every call (needed transitively for kernel32, free on linux); no shadow space (a Win64-only duty — it lives inside cap_* win64 bodies next to the kernel32 calls); no red zone. Roles are SysV's (lineage in docs/abi-lineage.md), so linux realizations marshal near-zero and win64 realizations own the translation. Real registers, no macro skin — one convention in source, bijection intact. OS→nadir entries (_start, future callbacks like WndProc) arrive in the TARGET convention and must normalize/shim first (_start: and rsp,-16).
```
